# Contributing to Project Android Windows

Thank you for your interest in contributing! We welcome contributions from everyone.

## Code of Conduct

Please read our [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Project-Android-Windows.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit with clear messages: `git commit -m "Add feature: description"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Development Setup

### Prerequisites
- Android SDK
- Kotlin/Java Development Kit
- Git

### Building
```bash
# Clone the repository
git clone https://github.com/sukdeobhilsql-dot/Project-Android-Windows.git

# Navigate to project
cd Project-Android-Windows

# Build the project
./gradlew build
```

## Contribution Guidelines

### Code Style
- Follow Kotlin naming conventions
- Use meaningful variable names
- Write clean, readable code
- Add comments for complex logic

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove, etc.)
- Keep messages concise but informative

### Pull Requests
- One feature per PR
- Include description of changes
- Reference related issues
- Ensure all tests pass

## Reporting Issues

### Bug Reports
- Use a clear, descriptive title
- Describe the exact steps to reproduce
- Include expected vs actual behavior
- Attach screenshots if applicable

### Feature Requests
- Explain the use case
- Describe the expected behavior
- Provide examples if helpful

## Testing

- Write tests for new features
- Ensure existing tests pass
- Test on multiple Android versions
